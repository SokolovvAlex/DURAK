# API Документация Административной Панели

## Базовый URL
```
/admin
```

## Аутентификация

Все эндпоинты (кроме `/login` и `/logout`) требуют аутентификации через JWT токен в куках `durak_access_token`.

---

## Эндпоинты Аутентификации

### POST /admin/login
**Описание:** Вход в систему администратора

**Права доступа:** Не требуется

**Входные параметры (Body, JSON):**
```json
{
  "login": "string",
  "password": "string"
}
```

**Выходные данные:**
```json
{
  "access_token": "string",
  "token_type": "bearer",
  "user": {
    "id": 0,
    "tg_id": 0,
    "username": "string",
    "name": "string",
    "login": "string",
    "is_admin": true,
    "is_super_admin": false,
    "is_active": true,
    "created_at": "2025-01-27T00:00:00"
  }
}
```

**Примечание:** Токен устанавливается в HTTP-only куку `durak_access_token`

---

### POST /admin/logout
**Описание:** Выход из системы

**Права доступа:** Не требуется

**Входные параметры:** Нет

**Выходные данные:**
```json
{
  "message": "Успешный выход из системы"
}
```

---

## Эндпоинты Управления Пользователями

### GET /admin/users
**Описание:** Получить список всех пользователей

**Права доступа:** Администратор

**Query параметры:**

| Параметр      | Тип    | Описание           | Обязательный |
|---------------|--------|--------------------|--------------|
| skip          | int    | Пропустить N записей| Нет          |
| limit         | int    | Лимит записей      | Нет          |
| is_admin      | bool   | Фильтр по админам  | Нет          |
| is_active     | bool   | Фильтр активности  | Нет          |
| is_super_admin| bool   | Фильтр суперадминов| Нет          |

**Выходные данные:**
```json
[{
  "id": 0,
  "tg_id": 0,
  "username": "string",
  "name": "string",
  "balance": 0.0,
  "is_admin": false,
  "is_super_admin": false,
  "is_active": true,
  "created_at": "2025-01-27T00:00:00",
  "login": "string"
}]
```

---

**Описание:** Изменить данные пользователя

**Права доступа:** Администратор

**URL параметры:**

| Параметр | Тип | Описание |
|----------|-----|----------|


## Эндпоинты Транзакций

### GET /admin/transactions
**Описание:** Получить список транзакций

**Права доступа:** Администратор

**Query параметры:**

| Параметр | Тип    | Описание              | Обязательный |
|----------|--------|-----------------------|--------------|
| skip     | int    | Пропустить N записей  | Нет          |
| limit    | int    | Лимит записей         | Нет          |
| type     | string | Фильтр по типу        | Нет          |
| status   | string | Фильтр по статусу     | Нет          |

**Типы:** `deposit`, `withdraw`, `referral_reward`, `payout`, `loss`, `admin_adjust`  
**Статусы:** `pending`, `posted`, `failed`, `reversed`

**Выходные данные:**
```json
[{
  "id": 0,
  "user_id": 0,
  "type": "deposit",
  "amount": 100.0,
  "status": "posted",
  "merchant_order_id": "string",
  "plat_guid": "string",
  "plat_withdraw_id": "string",
  "created_at": "2025-01-27T00:00:00"
}]
```

---
